Geral Diferenças Exemplos

Request (definição na internet)

A Request ou requisição traduzindo diretamente para português, é o pedido que um cliente realiza a nosso servidor. Esse pedido contém uma série de dados que são usados para descrever exatamente o que o cliente precisa. Vamos pensar que um cliente precisa cadastrar um novo produto, ele deve passar todos os dados necessários para o cadastro acontecer de maneira correta, inclusive os dados que foram digitados pelo usuário em um formulário, no caso de uma aplicação web. No navegador toda vez que trocamos de página ou apertamos enter na barra de endereço uma nova request é feita. Independente se estamos apenas pedindo a exibição de uma página, cadastrando um novo recurso, atualizando ou excluindo.

Response

Vimos que o cliente envia uma Request (requisição) ao servidor. Essa requisição possui todas as informações acerca do que o cliente espera receber de volta. O servidor web ao receber essas informações precisa enviar uma resposta ao cliente, nesse ponto entra a Response. A Response (resposta) nada mais é do que a resposta que o servidor envia ao cliente. Essa resposta pode conter os dados que realmente o cliente esperava receber ou uma resposta informando que alguma coisa deu errado.

Passo a Passo

Request -> Server -> Response

Request

Linha de pedido

A linha de pedido é composta pelo identificador do metodo (GET,POST,ETC...), URI e a versão do protocolo (HTTP 0.9,1.0,1.1,1.2).

Cabeçalho

Já o cabeçalho é onde passamos as informações sobre o nosso request, nele pode haver inúmeros parâmetros, que irão determinar o comportamento do servidor com o nosso request. Podemos ver como exemplo o User-Agent, que informa qual navegador esta sendo feito este request, também temos como exemplo cookie, que passa as informações em cookie do navegador para o servidor.

Corpo

O corpo do request irá conter as informações que serão enviadas para o servidor, exemplo: Se meu request é um request de metodo POST, de um formulario, eu enviarei no corpo nome, telefone, email...

Response

Linha de Status

A linha de status contem a versão do protocolo HTTP, codigo de status e o texto de status

Cabeçalho

O cabeçalho da Response funciona da mesma forma do cabeçalho do request, trazendo informações adicionais sobre a response, como o tipo de dado como resposta, o nome do server, a data...

Corpo

Enfim, a parte mais importante da response o corpo, onde contem as informações que foram solicitadas no request, usando meu exemplo passado se eu requsitei entrar no site stackoverflow, a response conterá todo conteudo necessario para o navegador processar e me apresentar a pagina.
